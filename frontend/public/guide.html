1. 소개

AI Memory Agent는 대화 내용과 문서에서 중요한 정보를 자동으로 추출하고, 시맨틱 검색을 통해 필요한 정보를 빠르게 찾을 수 있는 지능형 메모리 관리 시스템입니다.

핵심 기능:
  • 대화에서 자동으로 중요한 메모리 추출
  • 문서 업로드 및 지식 베이스 구축
  • 시맨틱 검색으로 의미 기반 정보 검색
  • 다양한 범위(개인, 프로젝트, 부서, 에이전트)로 메모리 분류

2. 주요 기능

2.1 메모리 범위

범위 설명 접근 권한
개인 (Personal) 개인적인 메모리 소유자만
대화방 (Chatroom) 특정 대화방에서의 메모리 소유자만
프로젝트 (Project) 프로젝트 관련 메모리 프로젝트 멤버
부서 (Department) 부서 공유 메모리 같은 부서원
에이전트 (Agent) AI 에이전트가 생성한 메모리 소유자만

2.2 메모리 중요도

  • 높음 (High): 매우 중요한 정보
  • 중간 (Medium): 일반적인 중요도
  • 낮음 (Low): 덜 중요한 정보

3. 메모리 관리

3.1 메모리 생성

1 대화방에서 메모리 생성

대화방에서 /remember 명령어를 사용하여 현재 대화 내용을 메모리로 저장합니다.

/remember 이 프로젝트의 마감일은 3월 15일입니다
2 자동 메모리 추출

AI가 대화 내용을 분석하여 중요한 정보를 자동으로 메모리로 추출합니다.

3.2 메모리 조회

1 지식 센터 → 지식 목록

사이드바에서 "지식 센터" → "지식 목록"을 클릭합니다.

2 탭으로 필터링

전체, 개인, 대화방, 프로젝트, 부서, 에이전트, 문서 탭으로 필터링할 수 있습니다.

3 메모리 확장

메모리 카드의 화살표를 클릭하여 전체 내용을 확인할 수 있습니다.

3.3 메모리 수정 및 삭제

  • 메모리 카드의 삭제 버튼(휴지통 아이콘)을 클릭하여 삭제
  • 삭제 시 확인 대화상자가 표시됩니다

4. 문서 관리

4.1 문서 업로드

1 지식 센터 → 문서

사이드바에서 "지식 센터" → "문서"를 클릭합니다.

2 파일 업로드

지원하는 파일 형식: PDF, TXT

  • 업로드 버튼을 클릭하거나 파일을 드래그 앤 드롭
  • 최대 파일 크기: 10MB
3 자동 처리

업로드된 문서는 자동으로 텍스트 추출, 청킹, 벡터화됩니다.

💡 팁: 문서는 청크 단위로 나뉘어 저장되므로, 긴 문서도 효율적으로 검색할 수 있습니다.

4.2 문서 조회

1 지식 목록 → 문서 탭

지식 목록 페이지에서 "문서" 탭을 클릭합니다.

2 문서 정보 확인

각 문서 카드에서 다음 정보를 확인할 수 있습니다:

  • 문서명
  • 파일 타입 (PDF, TXT)
  • 파일 크기
  • 청크 수
  • 업로드 날짜
  • 처리 상태

5. 지식 검색

5.1 기본 검색

1 지식 센터 → 지식 검색

사이드바에서 "지식 센터" → "지식 검색"을 클릭합니다.

2 검색어 입력

검색창에 검색어를 입력합니다 (최소 2글자).

3 범위 필터링

검색 범위를 선택하여 결과를 필터링할 수 있습니다:

  • 전체: 모든 메모리와 문서 검색
  • 개인, 대화방, 프로젝트, 부서, 에이전트: 특정 범위의 메모리 검색
  • 문서: 문서만 검색

5.2 시맨틱 검색

AI Memory Agent는 키워드 매칭뿐만 아니라 의미 기반 검색을 제공합니다.

시맨틱 검색 예시:
  • "프로젝트 마감일" 검색 → "3월 15일에 프로젝트 완료" 메모리도 검색됨
  • "커피 선호" 검색 → "아메리카노를 좋아함" 메모리도 검색됨

5.3 검색 결과

  • 유사도 점수로 정렬 (높을수록 관련성 높음)
  • 메모리와 문서가 통합된 형태로 표시
  • 메모리는 바로 삭제 가능

6. 에이전트 활용

6.1 에이전트란?

에이전트는 AI Memory Agent SDK를 사용하여 외부 시스템과 통합할 수 있는 기능입니다. 에이전트를 통해 대화 내용을 자동으로 메모리로 저장할 수 있습니다.

6.2 에이전트 메모리 확인

1 지식 목록 → 에이전트 탭

지식 목록 페이지에서 "에이전트" 탭을 클릭합니다.

2 에이전트별 그룹화

에이전트 인스턴스별로 메모리가 그룹화되어 표시됩니다.

6.3 테스트 챗봇 사용

테스트 챗봇을 통해 에이전트 기능을 테스트할 수 있습니다.

python tests/chatbot/test_chatbot.py
💡 테스트 챗봇 명령어:
  • /exit: 챗봇 종료
  • /clear: 대화 기록 초기화
  • /memory: 대화에서 메모리 추출
  • /help: 도움말

7. 활용 팁

💡 효율적인 메모리 관리 팁:
  1. 정기적인 메모리 정리: 중요하지 않은 메모리는 삭제하여 검색 성능 유지
  2. 적절한 범위 설정: 공유가 필요한 메모리는 프로젝트나 부서 범위로 설정
  3. 중요도 표시: 중요한 메모리는 "높음"으로 설정하여 우선순위 관리
  4. 문서 활용: 자주 참고하는 문서는 업로드하여 지식 베이스 구축
  5. 검색어 최적화: 구체적인 검색어 사용으로 정확한 결과 얻기
⚠️ 주의사항:
  • 메모리는 삭제 후 복구할 수 없습니다
  • 대용량 문서 업로드 시 처리 시간이 소요될 수 있습니다
  • 민감한 정보는 개인 범위로 저장하세요

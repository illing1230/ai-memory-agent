1. 소개

AI Memory Agent SDK를 사용하면 외부 AI 에이전트(챗봇, 가상 비서전트 등)에 지능적인 메모리 관리 기능을 쉽게 통합할 수 있습니다. SDK를 통해 대화 내용을 자동으로 메모리로 저장하고, 필요할 때 검색하여 활용할 수 있습니다.

SDK 주요 기능:
  • 메모리, 메시지, 로그 데이터 전송
  • 동기 및 비동기 클라이언트 지원
  • 자동 재시도 및 에러 처리
  • 타입 안전한 API
  • 간편한 사용자 매핑

2. 사전 준비

2.1 필수 요구사항

  • Python 3.8+
  • AI Memory Agent 서버 접속 가능
  • Agent Instance API Key

2.2 Agent Instance 생성

1 AI Memory Agent 웹 대시보드 접속

브라우저에서 AI Memory Agent 대시보드에 접속합니다.

2 Agent Marketplace 이동

사이드바에서 "Agent" → "Marketplace"를 클릭합니다.

3 Agent Instance 생성

원하는 Agent Type을 선택하고 "Instance 생성" 버튼을 클릭합니다.

4 API Key 확인

생성된 Agent Instance의 API Key를 복사합니다. 이 API Key는 SDK 연동에 필요합니다.

⚠️ 중요: API Key는 안전하게 보관하세요. 유출 시 타인이 귀하의 Agent Instance에 접근할 수 있습니다.

5. SDK 연동

5.1 동기 클라이언트 사용

from ai_memory_agent_sdk import AIMemoryAgentSyncClient
from dotenv import load_dotenv

# .env 파일 로드
load_dotenv()

# 클라이언트 초기화
client = AIMemoryAgentSyncClient(
    api_key="your_api_key",
    base_url="http://localhost:8000",
    agent_id="your_agent_id"
)

# 메모리 전송
result = client.send_memory(
    content="사용자가 커피를 좋아합니다",
    metadata={
        "source": "chatbot",
        "category": "preference"
    }
)

print(f"메모리 ID: {result['id']}")

5.2 비동기 클라이언트 사용

import asyncio
from ai_memory_agent_sdk import AIMemoryAgentClient

async def main():
    # 클라이언트 초기화
    async with AIMemoryAgentClient(
        api_key="your_api_key",
        base_url="http://localhost:8000",
        agent_id="your_agent_id"
    ) as client:
        # 메모리 전송
        result = await client.send_memory(
            content="프로젝트 마감일은 3월 15일입니다",
            metadata={
                "source": "chatbot",
                "category": "fact",
                "importance": "high"
            }
        )
        print(f"메모리 ID: {result['id']}")

asyncio.run(main())

5.3 데이터 타입

SDK는 세 가지 데이터 타입을 지원합니다:

데이터 타입 메서드 용도
memory send_memory() 중요한 정보 저장 (벡터화됨)
message send_message() 대화 기록 저장
log send_log() 시스템 로그 저장

6. 사용 예시

6.1 챗봇에 메모리 기능 추가

from ai_memory_agent_sdk import AIMemoryAgentSyncClient

class Chatbot:
    def __init__(self, api_key: str, base_url: str, agent_id: str):
        self.client = AIMemoryAgentSyncClient(
            api_key=api_key,
            base_url=base_url,
            agent_id=agent_id
        )
    
    def process_message(self, user_input: str) -> str:
        # 사용자 메시지를 메시지로 저장
        self.client.send_message(
            content=user_input,
            metadata={"source": "chatbot"}
        )
        
        # AI 응답 생성
        ai_response = self.generate_response(user_input)
        
        # AI 응답을 메시지로 저장
        self.client.send_message(
            content=ai_response,
            metadata={"source": "chatbot"}
        )
        
        return ai_response
    
    def extract_memory(self, conversation: str) -> str:
        # 대화에서 중요한 정보 추출
        memory = self.analyze_conversation(conversation)
        
        # 추출된 메모리를 저장
        self.client.send_memory(
            content=memory,
            metadata={
                "source": "chatbot",
                "category": "preference",
                "importance": "medium"
            }
        )
        
        return memory

6.2 사용자 매핑

외부 시스템의 사용자 ID를 AI Memory Agent의 내부 사용자 ID와 매핑할 수 있습니다.

# 외부 사용자 ID로 메모리 전송
client.send_memory(
    content="사용자의 선호도 정보",
    external_user_id="user_12345",  # 외부 시스템의 사용자 ID
    metadata={
        "source": "external_system",
        "category": "preference"
    }
)

6.3 에러 처리

from ai_memory_agent_sdk import (
    AIMemoryAgentSyncClient,
    AuthenticationError,
    APIError,
    ConnectionError
)

try:
    client = AIMemoryAgentSyncClient(
        api_key="your_api_key",
        base_url="http://localhost:8000",
        agent_id="your_agent_id"
    )
    
    result = client.send_memory(
        content="테스트 메모리",
        metadata={"source": "test"}
    )
    
except AuthenticationError as e:
    print(f"인증 오류: {e}")
    
except APIError as e:
    print(f"API 오류: {e}")
    
except ConnectionError as e:
    print(f"연결 오류: {e}")
    
except Exception as e:
    print(f"알 수 없는 오류: {e}")

7.3 배치 처리

import asyncio
from ai_memory_agent_sdk import AIMemoryAgentClient

async def batch_send_memories(memories: list[str]):
    async with AIMemoryAgentClient(
        api_key="your_api_key",
        base_url="http://localhost:8000",
        agent_id="your_agent_id"
    ) as client:
        tasks = [
            client.send_memory(content=memory)
            for memory in memories
        ]
        results = await asyncio.gather(*tasks, return_exceptions=True)
        
        for i, result in enumerate(results):
            if isinstance(result, Exception):
                print(f"메모리 {i} 전송 실패: {result}")
            else:
                print(f"메모리 {i} 전송 성공: {result['id']}")

# 사용 예시
memories = [
    "사용자는 Python을 선호합니다",
    "프로젝트 마감일은 3월 15일입니다",
    "팀 규모는 5명입니다"
]

asyncio.run(batch_send_memories(memories))

8. 문제 해결

8.1 일반적인 문제

문제: 인증 오류 (401)
  • API Key가 올바른지 확인하세요
  • Agent Instance가 활성화되어 있는지 확인하세요
문제: 연결 오류
  • 서버 URL이 올바른지 확인하세요
  • 서버가 실행 중인지 확인하세요
  • 방화벽 설정을 확인하세요
문제: 타임아웃
  • 기본 타임아웃은 30초입니다
  • 클라이언트 초기화 시 timeout 파라미터로 조정 가능합니다

8.2 디버깅 팁

💡 디버깅 팁:
  1. 상세 로그 활성화: SDK 내부 로그를 확인하여 문제 진단
  2. 헬스 체크: health_check()로 서버 상태 확인
  3. 네트워크 테스트: curl로 API 직접 호출 테스트
  4. 메타데이터 검증: 메타데이터가 올바른 형식인지 확인

8.3 API 직접 테스트

# curl로 API 테스트
curl -X POST http://localhost:8000/api/v1/agents/your_agent_id/data \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your_api_key" \
  -d '{
    "data_type": "memory",
    "content": "테스트 메모리",
    "metadata": {"source": "test"}
  }'

9. 대화방 사용 가이드

9.1 대화방 생성

대화방을 생성하여 AI와 대화하고 메모리를 활용할 수 있습니다.

1 새 대화방 만들기

사이드바의 "대화방" 섹션에서 "+" 버튼을 클릭합니다.

2 대화방 정보 입력

대화방 이름과 유형(개인/프로젝트/부서)을 선택합니다.

3 대화방 생성

"생성" 버튼을 클릭하여 대화방을 만듭니다.

9.2 메모리 소스 설정

대화방에서 사용할 메모리 소스를 설정하여 AI가 더 정확한 답변을 제공할 수 있습니다.

1 메모리 소스 버튼 클릭

대화방 헤더의 "📦 메모리 소스" 텍스트를 클릭합니다.

2 메모리 소스 선택

다음 메모리 소스 중에서 선택할 수 있습니다:

  • 이 대화방: 현재 대화방의 메모리 사용
  • 다른 대화방: 다른 대화방의 메모리 사용
  • 개인 전체: 개인 모든 메모리 사용 (⚠️ 주의 필요)
  • 프로젝트: 특정 프로젝트의 메모리 사용
  • 부서: 특정 부서의 메모리 사용
3 설정 저장

"저장" 버튼을 클릭하여 설정을 적용합니다.

💡 팁: 메모리 소스를 적절히 설정하면 AI가 관련성 높은 정보를 더 빠르게 찾을 수 있습니다.

9.3 대화방 공유

대화방을 다른 사용자와 공유하여 협업할 수 있습니다.

1 공유 버튼 클릭

대화방 헤더의 공유 버튼을 클릭합니다.

2 공유 대상 선택

공유할 사용자 또는 그룹을 선택합니다.

3 권한 설정

공유 대상의 권한을 설정합니다:

  • 멤버: 메시지 전송 및 메모리 저장 가능
  • 조회자: 메모리만 조회 가능
4 공유 완료

"공유" 버튼을 클릭하여 공유를 완료합니다.

10. 지식 센터 사용 가이드

10.1 메모리 검색

저장된 메모리를 검색하여 필요한 정보를 빠르게 찾을 수 있습니다.

1 검색 페이지 이동

사이드바에서 "지식 센터" → "검색"을 클릭합니다.

2 검색어 입력

검색창에 찾고자 하는 키워드를 입력합니다.

3 필터 적용

필요한 경우 다음 필터를 적용할 수 있습니다:

  • 데이터 타입: 메모리, 메시지, 로그
  • 범위: 개인, 프로젝트, 부서, 대화방
  • 기간: 특정 기간 내의 메모리
4 검색 결과 확인

검색 결과를 확인하고 필요한 메모리를 클릭하여 상세 내용을 봅니다.

💡 팁: 시맨틱 검색을 지원하므로 정확한 키워드가 아니어도 관련 내용을 찾을 수 있습니다.

10.2 메모리 관리

저장된 메모리를 조회하고 삭제할 수 있습니다.

1 지식 목록 페이지 이동

사이드바에서 "지식 센터" → "지식 목록"을 클릭합니다.

2 메모리 목록 확인

저장된 모든 메모리 목록을 확인합니다.

3 메모리 삭제

삭제할 메모리의 휴지통 아이콘을 클릭합니다.

⚠️ 주의: 삭제된 메모리는 복구할 수 없습니다. 신중하게 삭제하세요.

10.3 문서 업로드

PDF, TXT 파일을 업로드하여 문서 내용을 메모리로 활용할 수 있습니다.

1 문서 업로드 페이지 이동

사이드바에서 "지식 센터" → "문서 업로드"를 클릭합니다.

2 파일 선택

"파일 선택" 버튼을 클릭하여 업로드할 파일을 선택합니다.

3 파일 업로드

"업로드" 버튼을 클릭하여 파일을 업로드합니다.

4 처리 완료 확인

파일이 청크로 분할되고 벡터화됩니다. 처리가 완료되면 "완료" 상태로 표시됩니다.

지원 파일 형식:
  • PDF (.pdf)
  • 텍스트 파일 (.txt)
  • 최대 파일 크기: 50MB

10.4 문서 미리보기

업로드된 문서를 미리보기하여 내용을 확인할 수 있습니다.

1 문서 목록 확인

문서 업로드 페이지에서 업로드된 문서 목록을 확인합니다.

2 문서 클릭

미리보기할 문서를 클릭하거나 눈(Eye) 아이콘을 클릭합니다.

3 문서 내용 확인

우측 패널에서 문서 내용을 청크별로 확인할 수 있습니다.

4 패널 닫기

X 버튼을 클릭하여 미리보기 패널을 닫습니다.

11. 권한 관리 사용 가이드

11.1 지식 권한

메모리의 접근 권한을 관리하여 데이터 보안을 유지할 수 있습니다.

1 지식 권한 페이지 이동

사이드바에서 "권한 관리" → "지식 권한"을 클릭합니다.

2 메모리 선택

권한을 설정할 메모리를 선택합니다.

3 권한 설정

다음 권한을 설정할 수 있습니다:

  • 읽기: 메모리 조회 권한
  • 쓰기: 메모리 수정 권한
  • 삭제: 메모리 삭제 권한
4 접근 대상 설정

권한을 부여할 사용자 또는 그룹을 선택합니다.

5 권한 저장

"저장" 버튼을 클릭하여 권한 설정을 적용합니다.

11.2 프로젝트

프로젝트를 생성하고 관리하여 팀 협업을 효율화할 수 있습니다.

1 프로젝트 페이지 이동

사이드바에서 "권한 관리" → "프로젝트"를 클릭합니다.

2 프로젝트 생성

"새 프로젝트" 버튼을 클릭하고 프로젝트 정보를 입력합니다.

3 팀원 초대

프로젝트에 참여할 팀원을 초대합니다.

4 역할 할당

팀원에게 적절한 역할을 할당합니다:

  • 관리자: 모든 권한
  • 멤버: 메모리 읽기/쓰기
  • 조회자: 메모리 읽기만

11.3 공유 설정

메모리와 문서를 다른 사용자와 공유할 수 있습니다.

1 공유할 항목 선택

공유할 메모리 또는 문서를 선택합니다.

2 공유 버튼 클릭

공유 버튼(Share2 아이콘)을 클릭합니다.

3 공유 대상 선택

공유할 사용자 또는 그룹을 선택합니다.

4 권한 설정

공유 대상의 권한을 설정합니다.

5 공유 완료

"공유" 버튼을 클릭하여 공유를 완료합니다.

💡 팁: 공유된 항목은 "지식 권한" 페이지에서 관리할 수 있습니다.
